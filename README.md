# JetLag Team OS

This repository is the working knowledge base for JetLag's product development.

It keeps strategy, product context, PRDs, engineering plans, analytics definitions, customer notes, and team operating docs in one place so future AI sessions can work with shared context instead of starting cold.

## What This Repo Is For

- Keep product decisions discoverable
- Give engineering and analytics a consistent place for implementation context
- Store customer and research insights close to planning docs
- Make handoffs between product, design, engineering, and data easier

## Starting Points

- `CLAUDE.md`: top-level routing and repo guidance
- `product-development/CLAUDE.md`: product development index
- `product-development/feature-index.yaml`: master lookup for active JetLag features
- `product-development/product/strategy/`: business context, users, vision, and roadmap
- `product-development/product/product-context/`: system and product overviews

## Current Setup

The original Team OS example structure has been preserved, but the main guidance files now point to JetLag-specific product development work.

Assumption used for this setup: JetLag is a travel-focused product that helps users reduce jet lag with itinerary-aware recovery plans, reminders, and progress tracking.
