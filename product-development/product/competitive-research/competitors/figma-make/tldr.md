# Figma Make — TL;DR

## Overview
Figma's design-to-code AI feature. Converts Figma designs into working code.

## Key Strengths
- Native integration with Figma's design workflow — designers generate code without leaving their tool
- Leverages existing design files and components
- Strong collaboration features inherited from Figma (multiplayer, commenting, permissions)
- Enterprise features inherited from Figma (SSO, SOC 2, admin console)

## Key Weaknesses
- Code quality concerns, particularly for complex interactions and state management
- Limited to what can be inferred from static designs
- No backend generation
- No deployment pipeline
- Early and rapidly evolving

## ICP Overlap
Design teams, agencies, design-to-development handoff workflows. Adjacent competitor — different entry point (design-first vs prompt-first).

## Pricing
Included in Figma plans. Expected premium tier.

## Forge Differentiation
Forge starts from a prompt and generates full applications. Figma Make starts from a design and generates frontend code. Different workflows, different strengths. Risk is Figma expanding into full-app generation.

## Bottom Line
Adjacent, not direct. Different entry point (design-first vs prompt-first). The threat is Figma's massive design user base getting code generation "for free" as part of their existing tool.
