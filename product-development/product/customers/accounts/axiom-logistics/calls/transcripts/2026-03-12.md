## API Rate Limiting Discussion (5:30 - 18:45)

**Deepa Patel:** Let me start with the rate limiting. We're hitting rate limits every afternoon between 2 and 4 PM when our logistics teams are all running reports simultaneously. That's exactly when we need Forge most, and it's exactly when it falls over.

**Morgan Wu:** Can you walk me through the specific pattern? Are these generation calls, deployment calls, or a mix?

**Deepa Patel:** Mostly generation and preview calls. We have logistics coordinators across 6 regional offices who use the dashboards we've built in Forge. Between 2 and 4 PM Eastern, they're all pulling route optimization reports, refreshing inventory views, running end-of-day reconciliation. Our telemetry shows we spike to about 500 API calls per minute during that window.

**Morgan Wu:** 500 per minute — that's well above the current Enterprise tier limit of 200 per minute. I can see why you're getting throttled consistently.

**Raj Malhotra:** And the error handling isn't great on our end when the throttling kicks in. We get 429s with a generic retry-after header, but our dashboards just show a spinner and eventually time out. Our users don't know what's happening — they just think Forge is down.

**Hannah Stulberg:** That's a terrible experience. Morgan, what's the path forward here?

**Morgan Wu:** A few things we can do. Short term, I want to look at your API call patterns and see if there's any batching or caching we can help implement to reduce the raw call volume. Longer term, I think we need a dedicated capacity tier for accounts at your scale. I've been advocating for this internally — your data will help make the case.

**Deepa Patel:** I appreciate that, but I need a timeline. This isn't a nice-to-have. Our VP of Operations asked me last week why the logistics tools keep going down in the afternoon. When I said it was an API rate limit on our prototyping platform, his response was "then find a platform that can handle our scale."

**Hannah Stulberg:** Understood, Deepa. We're going to treat this as a P0. Morgan, can we get a concrete proposal to Deepa by end of next week?

**Morgan Wu:** Yes. I'll pull the usage data from their account this week and have a proposal for dedicated capacity by March 19.

## CI/CD Integration (18:45 - 32:00)

**Hannah Stulberg:** Raj, I know the CI/CD integration has been a major topic for you. Can you walk us through exactly what your team needs?

**Raj Malhotra:** Sure. We run Jenkins for everything — builds, tests, deploys, the whole lifecycle. Every tool our team ships goes through the same pipeline: code review, automated tests, staging deploy, smoke tests, production deploy. Our DevOps team refuses to adopt Forge for production deploys until we can plug it into our existing CI/CD pipeline. They don't want another manual step — they want it in the same Jenkins workflow as everything else.

**Alex Rivera:** That makes total sense. What does the ideal integration look like for you?

**Raj Malhotra:** A CLI or API that we can call from a Jenkins pipeline stage. Something like `forge deploy --project=dashboard-v2 --env=production` that returns a deployment URL and status. We also need the ability to roll back programmatically if our smoke tests fail.

**Alex Rivera:** Got it. And you mentioned you have 4 tools blocked right now?

**Raj Malhotra:** Four tools that are fully built and tested in Forge, sitting in limbo because we can't deploy them through our standard workflow. My team built them, they work great in preview, but we can't put them in front of users without going through the pipeline.

**Morgan Wu:** Raj, are you using GitHub Actions at all, or is it purely Jenkins?

**Raj Malhotra:** Jenkins for now, but we're evaluating GitHub Actions for some newer services. If you build a GitHub Actions integration first, that would be useful too — but Jenkins is the priority because that's where the 4 blocked tools live.

**Alex Rivera:** Fair. I'm going to scope this out as a phased approach — Jenkins first, then GitHub Actions. I'll have a scoping doc to you by March 26.

**Deepa Patel:** That would be great. And honestly, if you get CI/CD and rate limiting sorted, you'll have a very happy customer. We want to expand Forge usage to our data science team next quarter, but I can't justify that ask internally if the platform can't handle our current load.
