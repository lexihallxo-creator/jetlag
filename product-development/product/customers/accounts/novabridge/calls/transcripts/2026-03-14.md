## Multi-File Generation Quality (12:00 - 22:00)

**Hannah Stulberg:** James, let's dig into the multi-file generation topic. You've mentioned this before — what's the current state from your perspective?

**James Okoro:** When I generate a single-page app, Forge nails it almost every time. But the moment I need 3+ files with shared state and routing, the success rate drops significantly. I've learned to break things into phases — generate the layout first, then add routing, then add state. But Amara shouldn't have to learn those tricks to be productive.

**Jordan Kim:** Can you give me a specific example of a failure case?

**James Okoro:** Sure. Last week I described a React app with three pages — a dashboard, a settings page, and a user profile — using React Router for navigation and Context API for shared auth state. Forge generated all three page components, which was great. But the routing configuration was wrong — it used an older React Router API syntax. The Context provider was defined but never wrapped around the app. And the import paths between files were inconsistent — some used relative paths, some used aliases that weren't configured.

**Jordan Kim:** Was that with the default React template or a custom configuration?

**James Okoro:** Default React template. And here's the thing — when I regenerated each piece individually and described the imports explicitly, it worked fine. The model clearly knows how to write correct React Router code and Context API code. It's the orchestration across files that breaks down.

**Jordan Kim:** That's a really precise diagnosis. The multi-file orchestration layer is something we're actively improving. The model can handle individual file generation well, but maintaining consistency across file boundaries — import paths, shared type definitions, provider wrapping — that's the hard problem we're working on.

**James Okoro:** Here's what I want to see — I describe a three-page app with shared auth state and a database layer, and Forge generates all the files in one pass with the correct imports and routing. That's the promise. We're not there yet, but the foundation is solid.

**Jordan Kim:** That's exactly the target state. Would you be interested in joining our beta program for multi-file improvements? We'd love to have a power user like you testing early builds.

**James Okoro:** Definitely. Sign me up.

**Hannah Stulberg:** We'll get you added this week.
