## Company Context

Volta Energy is a renewable energy company with an 8-person engineering team running 6 active projects on Forge. They use Forge primarily for internal tool prototyping — energy monitoring dashboards, site configuration panels, and field operations interfaces. They built 6 tools last quarter but only deployed 2, primarily due to deployment infrastructure overhead.

## Key Themes

### 1. Inline Editing Breaks on Nested Layouts (Pain Point — P1)

Carlos built an energy grid monitoring dashboard with 6 nested chart components arranged in a complex grid-within-grid layout. When he attempts to inline-edit a single chart component, the layout of the surrounding components breaks — charts resize, padding collapses, and grid alignment shifts. He's found a workaround using component decomposition (editing each chart as an isolated component and then reassembling), but it's fragile and time-consuming. As Carlos put it: "I built an energy grid monitoring dashboard with 6 nested chart components. When I try to inline-edit one chart, the layout of the other 5 breaks. I end up regenerating the whole page and losing my tweaks." Jordan confirmed this is a known limitation of the inline editing engine's DOM diffing approach and discussed potential fixes.

### 2. Environment Variable Management for Multi-Site Deploys (Pain Point — P1)

Volta deploys identical tools to 8 different energy sites, each with unique API endpoints, monitoring credentials, and regional configuration parameters. Nina's team currently manages this by manually editing config files before each deployment — a process that's both error-prone and unscalable. Nina described it plainly: "We deploy the same tool to 8 different energy sites. Each site has different API endpoints and credentials. Right now we're manually editing config files for each deploy. It's begging for an environment variable system." This is blocking them from deploying more tools to production — it's a key reason only 2 of their 6 built tools have shipped.

### 3. Team Workspace Templates (Nice-to-Have — P2)

Nina mentioned that her team frequently starts new projects from similar foundations — a dashboard layout with their standard charting library, Volta branding, and common data connectors. She'd like the ability to save a project as a team template that any team member can instantiate. Lower priority than the editing and deployment issues, but would improve team velocity.
