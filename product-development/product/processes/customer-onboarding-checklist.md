# Customer Onboarding Checklist

Operational checklist for onboarding a new customer to the Forge platform. Used by Sales, PM, and Customer Success.

# Pre-Onboarding (Before Kickoff)

- [ ] Contract signed and billing set up in Stripe
- [ ] Customer record created in CRM with account owner assigned
- [ ] Onboarding kickoff meeting scheduled (within 3 business days of contract signing)
- [ ] Internal Slack channel created: `#customer-[company-name]`
- [ ] PM and engineering lead identified for the account

# Kickoff Meeting

- [ ] Introductions - customer team meets Forge team
- [ ] Walk through customer's primary use case and goals
- [ ] Agree on success criteria for the first 30 days
- [ ] Review pricing tier and included features
- [ ] Set up recurring check-in cadence (weekly for first month, then bi-weekly)

# Account Setup (Week 1)

- [ ] Admin account created for customer's primary contact
- [ ] Team workspace created with appropriate tier settings
- [ ] Team members invited via email or SSO (if Enterprise)
- [ ] Custom domain configured (if applicable)
- [ ] Billing confirmed - first invoice generated

# Enablement (Week 1-2)

- [ ] Live demo / training session for the customer's team (60 min)
- [ ] Share getting-started guide and help center links
- [ ] Walk through first generation and deploy end-to-end
- [ ] If Teams/Enterprise: set up shared component library with customer's brand assets
- [ ] Ensure at least 3 team members have completed their first generation

# Health Check (Week 3-4)

- [ ] Review usage metrics: generations, deploys, active users
- [ ] Check-in call: gather feedback, address blockers
- [ ] Identify power users and potential champions
- [ ] Document any feature requests in Linear / Jira / Asana (label: `customer-reported`)
- [ ] Confirm customer is on track to meet 30-day success criteria

# Ongoing

- [ ] Regular check-ins per agreed cadence
- [ ] Quarterly business review (QBR) for Teams and Enterprise
- [ ] Monitor health signals: login frequency, generation volume, support tickets
- [ ] Flag churn risk if usage drops below baseline for 2 consecutive weeks
