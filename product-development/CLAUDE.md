# Product Development

All JetLag product development artifacts live here: strategy, product, engineering, analytics, data engineering, and design references.

## Doc Index

| Path | Description |
|------|-------------|
| [product/CLAUDE.md](product/CLAUDE.md) | Product strategy, PRDs, customers, research, launches, and workflows |
| [engineering/CLAUDE.md](engineering/CLAUDE.md) | Engineering plans, RFCs, and bug investigations |
| [analytics/CLAUDE.md](analytics/CLAUDE.md) | Metrics, dashboards, event definitions, experiments, and investigations |

| [design/CLAUDE.md](design/CLAUDE.md) | Design references and links to source design work |
| `feature-index.yaml` | Master feature lookup for active JetLag initiatives |

## Recommended Workflow

1. Start with strategy or customer context before opening a new PRD.
2. Add or update the PRD in `product/PRDs/`.
3. Add linked implementation work in `engineering/plans/`.
4. Add instrumentation, success metrics, and dashboards in `analytics/`.
5. Update `feature-index.yaml` so the work is discoverable.
